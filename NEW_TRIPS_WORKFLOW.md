# Инструкция: как добавлять новые путешествия

Этот файл нужен, чтобы в новом окне не разбираться заново во всем проекте.

## Главное

- Текст берем из LiveJournal, а не пересказываем своими словами.
- Фотографии берем только из Google Drive по ссылкам.
- Локальные `jpg/png/webp` для новых путешествий не сохраняем в `content/`.
- Сначала идет текст, потом фотолента по смыслу.
- Если нужен новый видимый заголовок или описание, его лучше согласовать с владельцем дневника.

## Где что лежит

- Главная поездка: `content/europe/<country>/<trip-slug>/_index.md`
- Главы поездки: `content/europe/<country>/<trip-slug>/<chapter-slug>/index.md`
- Страны: `content/europe/<country>/_index.md`
- Места: `content/places/<place-slug>/_index.md`

Пример:

- поездка: `content/europe/germany/germany-2011/_index.md`
- глава: `content/europe/germany/germany-2011/overview/index.md`

## Что писать в поездке

В файле поездки `_index.md` обычно есть:

- `title`
- `description`
- `date`
- `trip_year`
- `cover_image`
- `geo_countries`

Важно:

- `geo_countries` нужны для раздела `По странам`
- одна поездка может входить сразу в несколько стран
- `cover_image` тоже должен быть Google Drive URL

## Что писать в главе

В файле главы `index.md` обычно есть:

- `title`
- `chapter_kind`
- `weight`
- `city`
- `places`
- `hero_image`
- `date`
- `auto_gallery: false`

Важно:

- `places` нужны для раздела `Места`
- `places` относятся к конкретной главе, а не ко всей поездке
- `auto_gallery: false` оставляем, чтобы не появлялись лишние автоматические галереи

## Правило стран и мест

- `По странам`: показываются полные путешествия
- `Места`: показываются отдельные главы, где это место реально есть

То есть:

- в поездке ставим `geo_countries`
- в главе ставим `places`

## Как вставлять фотографии

Для фотолент используем shortcode:

```md
{{< photoscroller items="URL1,URL2,URL3" >}}
```

Сейчас единый современный стиль фотолент применяется ко всем `photoscroller` автоматически.

Правила:

- сначала фото с людьми, потом остальные
- ленты разбиваем по смыслу рассказа
- если фото одно, можно использовать обычный `photo`, а не ленту
- если короткая лента не скроллится, стрелки должны скрываться автоматически

## Формат URL фото

Используем Google Drive thumbnail URL, например:

```text
https://drive.google.com/thumbnail?id=FILE_ID&sz=w1600
```

Для обложек можно использовать `w2000`.

## Как добавлять новое место

Если глава должна попадать в `Места`, создаем файл:

`content/places/<place-slug>/_index.md`

Минимально:

```md
---
title: "Köln - Кёльн, Германия"
---
```

Рекомендация по стилю названия:

- сначала локальное название
- потом русское
- потом страна

Пример:

- `Madrid - Мадрид, Испания`
- `Köln - Кёльн, Германия`
- `Lauterbrunnen - Лаутербруннен, Швейцария`

## Локальная проверка

Запускать только так:

```powershell
npm run dev
```

Почему:

- этот режим уже зафиксирован как локальный development
- он использует `renderToMemory`
- не должен подмешивать production-страницы из `public`

Если локально все выглядит без стилей:

1. остановить старый `hugo`
2. снова запустить `npm run dev`
3. открыть `http://127.0.0.1:1313/`
4. сделать `Ctrl+F5`

## Production

Production-сборка:

```powershell
npm run build
```

GitHub Pages использует:

- `hugo.yaml`
- `hugo.prod.yaml`

## Перед завершением работы

Нужно проверить:

- страница главы открывается локально
- фото идут из Google Drive
- нет локальных картинок в `content/`
- поездка попадает в нужные страны
- глава попадает в нужные `Места`
- `npm run build` проходит

## Короткий чеклист для нового путешествия

1. Взять текст из LJ.
2. Найти/создать поездку `_index.md`.
3. Найти/создать главу `index.md`.
4. Разложить фото из Google Drive на смысловые ленты.
5. Проставить `places` в главе.
6. Проставить `geo_countries` в поездке.
7. При необходимости создать новые `content/places/.../_index.md`.
8. Проверить локально через `npm run dev`.
9. Проверить production-сборку через `npm run build`.
